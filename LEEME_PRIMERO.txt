🎯 ¿QUÉ HACER AHORA?
═══════════════════════════════════════════════════════════════════════════════

PASO 1: Subir a GitHub
└── Opción A: Usa GitHub Desktop (más fácil)
    https://desktop.github.com/

└── Opción B: Usa Git desde la terminal
    git init
    git add .
    git commit -m "Initial commit"
    git remote add origin https://github.com/TU-USUARIO/krpano-sync.git
    git push -u origin main


PASO 2: Desplegar en Railway
└── 1. Ve a: https://railway.app/
└── 2. Login con GitHub
└── 3. New Project → Deploy from GitHub repo
└── 4. Selecciona tu repositorio
└── 5. Espera 2-3 minutos
└── 6. Settings → Generate Domain
└── 7. Copia tu URL: https://tu-proyecto.up.railway.app


PASO 3: Configurar tour.html
└── Abre tour.html (en tu proyecto Krpano)
└── Busca línea 98 y cambia:
    const SERVER_URL = 'https://tu-proyecto.up.railway.app';


PASO 4: Subir a Hostinger
└── Sube a public_html/:
    - tour.html (modificado)
    - krpano.xml
    - krpano.js
    - video/
    - plugins/
    - skin/

└── Sube a public_html/admin/:
    - admin.html (de la carpeta github/public/)

🆘 ¿PROBLEMAS?
═══════════════════════════════════════════════════════════════════════════════

1. Los clientes no conectan (indicador rojo):
   └── Verifica que la URL en tour.html sea correcta
   └── Usa HTTPS (no HTTP)
   └── Copia la URL exacta de Railway

2. El servidor no despliega:
   └── Revisa los logs en Railway (Deployments → último deployment)
   └── Verifica que todos los archivos estén subidos a GitHub

3. "This site can't be reached":
   └── Genera el dominio en Railway (Settings → Generate Domain)
   └── Espera 1-2 minutos después de generar
